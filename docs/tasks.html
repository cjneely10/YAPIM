Task¶

Task provides the primary API methods with which users will interact.

class yapim.tasks.task.Task(record_id: str, task_scope: str, config_manager: yapim.utils.config_manager.ConfigManager, input_data: dict, added_data: dict, wdir: str, display_messages: bool)¶

The Task class manages completing a set of instructions for an item in the input set. The Task class operates on a single input. str/Path objects that are defined by self.output will be validated for existence after a Task completes its run() invocation. If the condition() method is overwritten, it will be called prior to the run() method.

create_script(cmd: Union[str, plumbum.machines.local.LocalCommand, List[Union[plumbum.machines.local.LocalCommand, str]]], file_name: str, parallelize: bool = False) → plumbum.machines.local.LocalCommand¶

Write a command to file and return its value packaged as a LocalCommand.

This is highly useful when incorporating programs that only launch in the directory in which it was called

Example:

script = self.create_script(self.local[“ls”][“~”], “cd.sh”)

This will create a file within self.wdir named cd.sh, the contents of which will be:

#!/bin/bash cd <wdir> || return

ls ~

This can then be run in parallel or singly: self.parallel(script) self.single(script)

This may also be used to parallelize operations at the Task level:

script = self.create_script(

[self.local[“ls”][“~”], self.local[“ls”][“~”], self.local[“ls”][“~”]], “cd.sh”, parallelize=True

)

This will create a file within self.wdir named cd.sh, the contents of which will be:

#!/bin/bash

cd <wdir> || return

ls ~ &

ls ~ &

ls ~ &

wait

parallel(cmd: plumbum.machines.local.LocalCommand, time_override: Optional[str] = None, threads_override: Optional[str] = None)¶

Launch a command that uses multiple threads This method will call a given command on a SLURM cluster automatically (if requested by the user)

A time-override may be specified to manually set the maximum time limit a command (job) may run on a cluster, which will override the time that is specified by the user in a config file. This is also possible for a thread count.

The command string will be written to the EukMetaSanity pipeline output file and will be printed to screen

Example: self.parallel(self.local[“pwd”])
